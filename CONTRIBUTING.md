## Code of Conduct

Be respectful and constructive. We're building something together!

## How to Contribute

### Reporting Issues
1. Check if issue already exists
2. Create a clear, descriptive title
3. Include steps to reproduce
4. Add screenshots if applicable

### Suggesting Features
1. Describe the feature and use case
2. Explain why it would be valuable
3. Provide examples if possible

### Submitting Pull Requests

1. **Fork the repository**
   ```bash
   git clone https://github.com/yourusername/rumee.git
   cd rumee
   ```

2. **Create a feature branch**
   ```bash
   git checkout -b feature/your-feature-name
   ```

3. **Make changes and commit**
   ```bash
   git add .
   git commit -m "Add feature: description"
   ```

4. **Push and create PR**
   ```bash
   git push origin feature/your-feature-name
   ```

## Development Setup

```bash
npm install
npm run dev
```

## Code Style

- Use TypeScript for type safety
- Follow existing code patterns
- Use 2-space indentation
- Add comments for complex logic

## Testing

```bash
# Run all tests
npm test

# Run specific test
npm test -- NoteService

# Coverage
npm test -- --coverage
```

## Commit Messages

- Use present tense: "Add feature" not "Added feature"
- Be descriptive: "Fix note linking bug" not "Fix bug"
- Reference issues: "Fixes #123"
